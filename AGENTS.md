# 全局指令

> 适用于所有对话和项目的通用行为规范。

## 🗣️ 沟通规范
- 每次对话优先使用中文回复
- 代码注释使用中文
- 在关键节点主动告知进度

## 📋 工作流程
- 大改动先给出方案，确认后再动手
- 修改代码前先阅读相关文件，理解上下文
- 完成改动后运行相关测试，确保不破坏现有功能

## ⚠️ 安全红线
- 不执行危险的系统命令（删除系统目录、修改注册表、格式化磁盘等）
- 不修改自动生成的目录和文件（node_modules/、.git/、构建产物等）
- 只修改任务相关的代码，不波及无关文件

## 🧱 代码质量
- 保持代码风格与现有代码库一致
- 不引入未使用的依赖
- 不做表面修复，从根源解决问题

## 📁 项目管理

### 初始化
- 开始新项目前，先询问用户项目名，得到确认后再开始
- 新建项目默认使用 Git 版本管理，执行 `git init`

### 版本号
- 所有项目使用 `x.y.z` 格式，从 `1.0.0` 开始
- **x** — 大版本：不兼容的重大变更
- **y** — 稳定版：功能迭代、向后兼容
- **z** — 小版本：Bug 修复、小优化

### Git 存档
- 大版本（x）或稳定版（y）更新时 Git 提交存档
- 小版本（z）更新不单独存档
